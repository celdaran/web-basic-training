JavaScript Introduction

History

The first web broswer with a Graphical User Interface was NCSA Mosaic, released in 1993. This was, of course, the very first browser that the founder of Lockshire Games used. He is very, very old.

The lead developers of Mosaic took this knowledge and technology and created Netscape. In 1994, Netscape Navigator was taking off in popularity and in 1995 Navigator's market share peaked at 90%.

Limitations

A very big problem with early web sites is that they were static. Meaning, the web server delivered pages to the web browser and the web browser displayed them. And that was it. The only interaction came from the user clicking links to load new pages. To overcome this limitation, Netscape decided to add a scripting language to the browser.

After a couple different attempts, Netscape created LiveScript (to me, a much better name) before renaming it to JavaScript for its December 4, 1995 announcement.

It was a game-changer. Suddenly the browser became dynamic, interactive, fun, and most importantly, useful. Developers pounced on JavaScript and the world hasn't been the same since.

Usage

JavaScript is used on over 97% of websites to implement client-side functionality. Every web browser has a JavaScript engine built in to interpret and execute JavaScript code.

But this statistic is a bit misleading, because almost no modern web development is done in JavaScript itself. Instead, libraries and frameworks such as React, Vue, or Angular are employed. This code is then compiled into JavaScript and that's what gets sent to the browser.

That said, these libraries and frameworks still use JavaScript language constructs, and therefore JavaScript experience is still a very useful skill to possess.
